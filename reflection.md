# Reflection

The process of translating system requirements into use case diagrams, detailed use case specifications, and corresponding test cases for the PRASA Smart Digital Monitoring System proved to be both intellectually demanding and highly informative. One of the primary challenges encountered during this assignment was ensuring consistency between the functional requirements identified in the previous phase and the use cases developed in this stage. This required careful interpretation and alignment to ensure that each use case accurately represented system functionality while remaining consistent with stakeholder expectations and operational objectives.

A further challenge involved determining the appropriate level of detail required within the use case specifications. Initially, there was uncertainty regarding how extensively the basic and alternative flows should be elaborated. However, it became evident that clarity and precision were essential in describing each step of system behavior. At the same time, it was necessary to avoid excessive detail that could obscure the overall logic of the system. The incorporation of alternative flows, particularly those addressing exceptional scenarios such as sensor malfunctions and network failures, was especially valuable. These scenarios facilitated a deeper understanding of how the system should respond under abnormal conditions, thereby contributing to the design of a more robust and resilient system.

The development of the use case diagram required a comprehensive understanding of the various actors interacting with the system. Identifying and categorizing actors, including control room operators, maintenance technicians, and IoT sensors, was crucial for accurately representing system interactions. Moreover, the application of relationships such as «include» and «extend» required logical reasoning to ensure that dependencies between use cases were correctly modeled. This process enhanced the structural clarity of the system and supported a more organized representation of its functional components.

The formulation of test cases further reinforced the understanding of system requirements. Each test case was explicitly linked to a corresponding requirement, ensuring traceability and facilitating effective validation of system functionality. The process of defining clear and measurable expected results, particularly for non-functional requirements such as performance and security, highlighted the importance of specifying system quality attributes. For instance, the inclusion of response time benchmarks underscored the necessity for efficiency in real-time monitoring systems, where timely data processing is critical.

In addition to the system design and testing components, Assignment 7 introduced the practical application of Agile project management using GitHub Projects. One of the main challenges was selecting the most suitable project template, as each template offers different structures and levels of automation. Evaluating templates such as Basic Kanban, Automated Kanban, and Bug Triage required an understanding of how each supports workflow management and Agile principles.

Customizing the Kanban board also presented some challenges. Adding new columns such as "Testing" and "Blocked" required careful consideration to ensure that they accurately reflected the workflow of the PRASA system. Linking issues derived from user stories and system requirements to the Kanban board required attention to detail to maintain consistency and traceability. Assigning tasks and organizing them across different workflow stages further reinforced the importance of structured task management.

Comparing GitHub Projects to other tools such as Trello and Jira provided additional insights. GitHub Projects is highly effective for software development due to its direct integration with repositories, making it easier to manage code and tasks in one environment. However, Trello offers a more user-friendly interface for beginners, while Jira provides more advanced features for large-scale enterprise project management.

Overall, these assignments provided valuable practical insight into both software engineering and Agile project management principles. They demonstrated how theoretical requirements can be systematically transformed into structured models, validation mechanisms, and managed workflows. Furthermore, they emphasized the importance of maintaining accuracy, clarity, and consistency throughout the system development lifecycle.

In conclusion, the knowledge and skills gained from these assignments will be highly beneficial in future software development projects. They have strengthened competencies in system analysis, design, testing, and Agile project management, while reinforcing the importance of a methodical and detail-oriented approach to developing reliable and efficient software systems.
