# Assignment 15 – Cross Project Contributions & Collaborative Development

## Overview

This assignment focused on contributing to peer software projects through GitHub Pull Requests. The objective was to gain practical experience in collaborative software development by identifying improvements, implementing features, fixing issues, and following professional GitHub workflows.

The contributions were completed across multiple repositories and successfully merged after review.

---

* Contribute to peer repositories.
* Implement meaningful software improvements.

* Improve existing software systems through feature enhancements and validation improvements.

## GitHub Workflow Followed

1. Fork repository.
2. Clone repository locally.
3. Create a feature branch.
4. Implement changes.
5. Commit code with descriptive messages.
6. Push changes to GitHub.
7. Create Pull Request.
8. Address review feedback.
9. Merge approved Pull Request.
