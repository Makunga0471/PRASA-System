# Domain Model

  ------------------------------------------------------------------------------
  Entity        Attributes          Methods             Relationships
  ------------- ------------------- ------------------- ------------------------
  Train         trainId, status,    start(), stop(),    Connected to Track,
                location            updateLocation()    monitored by Sensor

  Track         trackId, condition, updateCondition()   Used by Train, monitored
                location                                by Sensor

  Sensor        sensorId, type,     detect(),           Installed on Track,
                status              sendData()          generates Alert

  Alert         alertId, message,   triggerAlert(),     Generated by Sensor,
                severity, timestamp notifyOperator()    handled by Operator

  Operator      operatorId, name,   monitorSystem(),    Manages Alerts
                role                respondToAlert()    

  Maintenance   maintenanceId,      scheduleRepair(),   Repairs Track/Train
                schedule, status    performRepair()     after Alert
  ------------------------------------------------------------------------------

## Business Rules

-   A Train must be assigned to one Track at a time\
-   A Track can have multiple Sensors\
-   A Sensor can generate multiple Alerts\
-   An Operator can handle multiple Alerts\
-   Maintenance is triggered by critical Alerts
